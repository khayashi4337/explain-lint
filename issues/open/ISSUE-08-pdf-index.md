# ISSUE-08 PDF対応 + 索引（back-of-book index）生成

- **深刻度**: 機能要望（low priority）
- **分類**: 機能拡張
- **branch**: feature/08-pdf-index（着手時）

## 背景

書籍の巻末によくある「索引（index / back-of-book index）」——用語が五十音順に並び、対応するページ番号が書いてあるページ。現在の explain-lint は Markdown テキストファイルのみ対応で、ページ番号の概念がない。

## 要件

1. **PDF入力対応**: PDFファイルを入力として受け取り、テキストを抽出して用語検出の対象にする。
2. **ページ番号の記録**: 台帳の `first_seen` 列に `book.pdf:p42 §Geometry` のようにファイル名・ページ番号・直近見出しを記録する。
3. **索引出力**: 台帳から「用語 → ページ番号」の索引を生成するコマンド（例: `--index`）。書籍の巻末索引と同様の形式で出力する。

## 設計上の考慮

- **依存関係**: PDFテキスト抽出には外部ライブラリ（`pypdf` / `pdfplumber` 等）が必要。現在のコアは「依存関係なし」が売りなので、PDF抽出はオプション機能として分離し、インストール時に選択可能にする（例: `pip install explain-lint[pdf]`）。
- **ページ境界の検出**: 抽出ライブラリが提供するページ番号をそのまま使うか、PDF内のページ番号表示と整合させるかの判断が必要。
- **索引のソート**: 日本語用語は五十音順、ラテン用語はアルファベット順の混在ソート。
- **複数ファイル**: 現在の複数ファイル対応（章ごとに分割された本など）を踏襲し、ファイル名:ページ番号の形式で出力。

## 完了条件

- PDFファイルを入力として用語検出ができること。
- 台帳にページ番号が記録されること。
- `--index` で索引形式の出力が得られること。
- 既存の Markdown 入力に対する動作が回帰しないこと。
