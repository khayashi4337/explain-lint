# ISSUE-09 日本語漢字・ひらがな用語の抽出（形態素解析）

- **深刻度**: 機能要望（low priority）
- **分類**: 機能拡張
- **branch**: feature/09-ja-morphological（着手時）

## 背景

現在の explain-lint は日本語のうちカタカナ（3文字以上）のみを抽出対象としている。
漢字・ひらがなを含む用語（例: `量子力学`、`非線形効果`、`固有値`）は抽出されない。
日本語の技術文書では漢字を含む専門用語が多数使われるため、実用上の大きなギャップがある。

## 要件

1. **形態素解析による単語抽出**: 日本語テキストを形態素解析し、名詞（固有名詞・一般名詞・複合名詞）を候補用語として抽出する。
2. **既存のカタカナ抽出との統合**: 現在の `KATAKANA` 正規表現による抽出と併用可能にする。
3. **フィルタリング方針**: 品詞（名詞のみ？固有名詞のみ？）、最小文字数、ストップワード（`もの`、`こと` 等の一般名詞）の検討。

## 設計上の考慮

- **依存関係**: 形態素解析には辞書データを含む外部ライブラリが必要。現在のコアは「依存関係なし」が売りなので、オプション機能として分離する（例: `pip install explain-lint[ja]`）。
- **候補ライブラリ**:
  - **MeCab** (fugashi) — 高速、辞書（IPA辞書等）が必要
  - **Janome** — 純Python、インストール簡単だがMeCabより遅い
  - **SudachiPy** — 形態素解析器SudachiのPython版
- **複合名詞の扱い**: `量子力学` を1用語とするか `量子` + `力学` に分割するか。形態素解析器の複合名詞抽出機能（MeCabの `--node-format` 等）を活用。
- **カタカナとの重複**: カタカナ用語は形態素解析でも抽出される可能性がある。既存の `KATAKANA` 正規表現との重複を避ける方針が必要。
- **パフォーマンス**: 形態素解析は正規表現より重いため、大規模文書での実行時間に注意。

## 完了条件

- 漢字・ひらがなを含む用語（例: `量子力学`、`固有値`）が抽出されること。
- 既存のカタカナ抽出・ラテン文字抽出が回帰しないこと。
- オプションインストール時のみ形態素解析が有効になり、コアの「依存関係なし」が維持されること。
- ストップワード（一般名詞のノイズ）が適切にフィルタリングされること。
