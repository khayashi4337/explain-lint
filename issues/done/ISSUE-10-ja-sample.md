# ISSUE-10 日本語サンプルの追加

- **深刻度**: 機能要望（low priority）
- **分類**: ドキュメント・サンプル
- **branch**: feature/10-ja-sample（着手時）

## 背景

現在のサンプル（`examples/sample.md`）は英語ベースで、カタカナ用語が2つ（`オブザーバブル`、`コヒーレンス`）含まれるのみ。日本語の技術文書における explain-lint の使い勝手を示すサンプルがない。

ISSUE-09（形態素解析による漢字・ひらがな用語抽出）が実装された後、日本語サンプルの価値がより高まる。しかし現状でもカタカナ抽出のデモとしては有用なため、先行して追加してもよい。

## 要件

1. **日本語サンプルファイル**: `examples/sample_ja.md` を作成し、日本語の技術文書風の文章を記載する。
2. **説明ギャップのデモ**: 英語サンプルと同様に、初出で説明されている用語とされていない用語を混在させる。
3. **カタカナ・漢字・ひらがなの混在**: 現状はカタカナのみ抽出対象だが、将来的な形態素解析対応（ISSUE-09）を見据え、漢字・ひらがなを含む用語も文章中に盛り込む。
4. **READMEからのリンク**: README.md / README_ja.md のサンプル参照を更新する。

## 設計上の考慮

- **ドメイン**: 物理学・数学など、既存サンプルと同分野が親和性が高い。あるいはソフトウェア工学など別分野でも可。
- **用語例**:
  - カタカナ（現状で抽出可能）: `オブザーバブル`、`コヒーレンス`、`ハミルトニアン`
  - 漢字（ISSUE-09実装後に抽出可能）: `量子力学`、`固有値`、`非線形効果`
- **台帳サンプル**: `examples/sample_ja.terms.md` も併せて作成し、判定済みの台帳例を示す。

## 完了条件

- `examples/sample_ja.md` が作成されていること。
- 英語サンプルと同等の構成（説明あり・なしの用語が混在）であること。
- カタカナ用語が現状のツールで抽出できること（`python -m explain_lint examples/sample_ja.md --dump` で確認）。
- README / README_ja.md からサンプルへの言及があること。
