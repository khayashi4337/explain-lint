# ISSUE-13: レビュー3回イテレートのスキップ（ISSUE-08〜11）

## 深刻度
低（プロセス違反）

## 概要
`issues/README.md` のワークフローでは、コミット時に3回・PR時に3回のレビューを
行うことが定められている。しかしISSUE-08〜11の実装時、このレビューをスキップして
コミット・マージを行ってしまった。

レビューは事後（マージ後）に実施し、3件の是正を行った:
1. ISSUE-09 PR 2回目: MCP `record_judgment` ツールに `use_morph`/`min_kanji` 不足 → 追加
2. ISSUE-11 コミット2回目: `test_pdf_input_without_pypdf` が実装をテストしていなかった → 修正
3. ISSUE-11 コミット3回目: 未使用の `_make_pdf` スタブと無駄なインポート → 削除

## 根本原因
- 実装→テスト→コミット→マージの流れが早く、レビューステップを忘れた
- チェックリストやガードレールがなかった

## 修正方針
- 今後はコミット前に必ず3回レビューを実施する
- ワークフローのリマインダーを強化（必要ならチェックリストを `issues/README.md` に明記）

## branch
（プロセス改善のため、コード変更なし。既に是正済み）
