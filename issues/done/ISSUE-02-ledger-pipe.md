# ISSUE-02 セル内 `|` で台帳の行が消失する（サイレントなデータ喪失）

- **深刻度**: 高（MOVEDより悪質＝データ喪失＋無限再判定）
- **分類**: 正当性
- **branch**: fix/02-ledger-pipe

## 症状
見出しに `|` を含む語を判定して台帳に書くと、その行が次回 `read_ledger` で**再パースに失敗して消える**。消えた語は次回 NEW 扱い→再判定→また書いて消える、の無限ループ。

## 根本原因
`write_ledger` の `esc` は `|`→`\|` にエスケープするが、`read_ledger` の `LEDGER_ROW` は `[^|]` で列分割するため、`\|` 内の生 `|` で列がずれて行マッチに失敗。**エスケープ（write）とアンエスケープ（read）が非対称**。`first_seen` が `§Heading | with pipe` の形になると発生。

## 修正方針
write/read のエスケープを対称化する。案:
- read 側で `\|` を一時トークンに退避→`|` で列分割→復元、write 側と厳密に対応させる。
- あるいはセル値に含まれうる `|`・改行・`\` をまとめてサニタイズ/エスケープする関数を1つ用意し write/read 双方が使う。
- セル内改行（`write_ledger` が未処理）も同時に対処。

## 完了条件（回帰テスト）
- `test_ledger_roundtrip`: term/notes/heading/first_seen に `|`・`\`・（可能なら）改行 を含むケースで **write→read が恒等**（行数・各セル値が一致）。
- 見出しに `|` を含む実ファイルで record_judgment→read で行が消えないこと。
